# Contributing

Thanks for improving this project.

## Before you open a pull request

- Keep the scripts beginner-friendly.
- Preserve config backups before changing user files.
- Never commit real keys, tokens, or private endpoints.
- Test the changed script on the target platform whenever possible.
- Update `README.md`, `README_EN.md`, and release notes when behavior changes.

## Local checklist

- Verify shell / PowerShell syntax.
- Verify prompts are easy for non-technical users to understand.
- Verify generated files only contain placeholders in the repository.
- Confirm no sensitive local paths or secrets are introduced.

## Pull request guidance

- Describe the user-visible improvement first.
- Include test notes for macOS or Windows.
- Mention whether the change affects GLM, Codex, or both.
