# Security Policy

If you find a security issue, please do **not** open a public issue with exploit details.

## Please report privately

Send a private report that includes:

- What is affected
- How it can be reproduced
- Possible impact
- Suggested mitigation if you have one

## Sensitive data rules

- Never publish real API keys in issues, pull requests, or release notes.
- Never commit generated local auth files such as `~/.codex/auth.json`.
- Never include customer-specific gateway URLs unless they are explicitly intended to be public.
