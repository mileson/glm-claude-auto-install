# v0.1.9

## 这次更新了什么

这次改进了 macOS 安装器在 npm 源不可访问时的提示。

现在如果用户遇到 `ENOTFOUND`、`getaddrinfo` 或 `registry.npmjs.org` 相关错误，安装器会先提示当前电脑连不上 npm 源，并给出可操作的处理步骤。

## 为什么这样改

少数网络环境下，npm 会因为 DNS、代理或 VPN 设置无法访问 npm 源。之前安装器会直接显示 npm 原始错误，对新手不够友好。

## 现在的体验

- 安装前会检查 npm 源是否可访问
- 会沿用用户当前配置的 npm registry
- 如果 npm 源不可访问，会提示用户检查网络、代理或 npm 代理配置
- 如果设置了可用镜像源，安装器会使用同一个源完成安装
